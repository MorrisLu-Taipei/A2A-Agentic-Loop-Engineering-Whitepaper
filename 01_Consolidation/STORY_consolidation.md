# 收斂故事:從「我發現了一個東西」到「我守得住哪一塊」

> 本檔是 ALE **收斂階段(Consolidation)的敘事**(Claude 代擬,作者可改寫第一人稱)。它記錄一個興奮的點子,如何經過外部研究的「冷水」,被誠實地收斂成可辯護的主題。對應盡職調查見 `../05_DueDiligence/ALE_PriorArt_RelatedWork_v2.md`。

---

## 一、興奮期:我好像發現了一個大東西

從發票案盤點出 **test collusion** 之後,有一陣子我是亢奮的。
「驗證 AI 的也是 AI,會一起錯」「換模型救不了,因為它們是相關估計器」——這些話講出來很有力,白皮書一路寫到 v2.3,愈寫愈像一個完整的框架。那時候我心裡偷偷想的是:**這會不會是個全新的東西?**

亢奮的時候最危險。因為**人會開始愛上自己的說法,而不是去查它是不是真的新。**

## 二、冷水:先去查「我是不是抄了別人」

在往下衝之前,我們做了一件不舒服但必要的事:**主動去查抄襲與先前技術。**
起因是看到一個名字很像的開源專案 `agenticloops-ai/agentic-ai-engineering`——「agentic」「loop」都撞字。如果要發 DOI,這種事必須先講清楚。

逐項比對後,結論是:**沒有抄襲。** 它是一套教人「怎麼建 agent」的教學集,我們談的是「agent 產出之後怎麼治理與驗證」,連要解的問題都不同,核心詞一個都沒重疊。

——但同一次搜尋,潑了我兩盆冷水:

1. **「Loop Engineering」早就是 2026 的通用詞了**(Data Science Dojo、Kilo、MindStudio 都在用,還有 agenticloops.ai 這個品牌)。我以為很潮的「Agentic Loop Engineering」,其實站在一塊已經很擁擠的地。
2. **多代理人 collusion 是一個活躍的研究領域**(好幾篇 arXiv)。我以為自己命名的現象,別人早從別的角度在談。

那一刻有點洩氣。但緊接著是更重要的領悟。

## 三、轉折:冷水裡有一塊浮木,而且是別人遞給我的

我仔細讀那些 collusion 文獻,讀到 **arXiv:2512.03097**(醫療多代理人共謀)時愣住了。

> 🔧 **誠實更正(v3.0)**:我第一次讀時把它的結論講反了,還寫進稿子——以為它證明「verifier 擋不住共謀」。**讀了 PDF 原文才發現相反**:多代理人**共識**確實不安全(攻擊成功率衝到 100%),但**接上外部可信指南的 verifier 把它壓到 0%**。這個錯是 Codex 的對抗式審查 + 我重讀原文才抓出來的(見 `../07_Lessons_Learned` L1)。

但弔詭的是:**正確的解讀更挺我。** 它真正說的是——「**模型共識 ≠ 安全;驗證的價值來自它接了外部證據,而非它也是另一個模型**」。這正是我要主張的:**從模型共識,轉向可執行證據。**

於是定位翻轉:
> 那些文獻不是我的對手,是我的**支持證據**(只要我**讀對**)。而我提供的不是「發現共謀」(別人做過),是**把它整合進完整 SDLC 的證據治理**。

## 四、收斂:誠實地讓出公共財,只留守得住的那塊

收斂的本質,是**承認哪些不是我的,然後把剩下真正屬於我的,講清楚。**

**讓出去(公共財/已有文獻,就地引用、不主張原創):**
- agent loop / **loop engineering(連 "Agentic Loop Engineering / ALE" 這個名字都被 arXiv:2509.06216 用過了 → 我們改名)** / guardrails
- 多代理人 collusion 這個現象;**連 code/test self-collusion 都已被 Code-A1 [arXiv:2603.15611] 提出 → 我們不主張命名/特化原創**
- mutation/property/metamorphic testing、ensemble 理論

**守得住(候選貢獻,整合 > 首創,待實證升級):**
1. 把「共享脈絡假通過」**整合進完整 Agentic SDLC 治理**(不只是 code/test 兩個模型的事)。
2. **以可執行證據取代模型共識**作 release gate(機械閘),輔以模型評議找分歧、人類審證據。
3. **技能資產化 + 去退化治理 + evidence repository + rollback**,串成可稽核、可回滾、可沉澱的生命週期。

然後收斂成一句話:
> *From model consensus to executable evidence — an evidence-governed lifecycle for agentic software development.*

並把問題收斂成 RQ1–RQ5,把每個主張標上證據等級——**我不再說「無懈可擊」,我說「這是候選貢獻,等 EXP-001 驗」。**

## 五、學到的事

審稿人最愛問兩句:「這不就是 DevOps 加 Agent?」「這別人不是做過了?」
收斂階段的工作,就是**在被問之前,自己先誠實回答完。**

> 真正的新穎性,不是「我發明了一切」,
> 而是「**我清楚知道哪些不是我的,所以你能相信剩下那塊真的是。**」

亢奮讓我以為自己很大;外部研究讓我變小;而收斂,讓我**站穩在一塊雖小、但守得住的地上**。
